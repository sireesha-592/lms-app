## PART 1 — DESKTOP .EXE APPS (Electron.js)

### Step 1 — Copy the conversion files into your project

Copy the entire `electron-wrapper/`, `android/`, `scripts/`, `docs/` folders
from this ZIP into your `lms-app/` root (next to `backend/` and `frontend/`).

Your folder should look like the structure shown above.

---

### Step 2 — Patch your backend server.js

Open `backend/server.js` and apply two changes described in `docs/server-patch.js`:

**Change 1 — CORS (around line 20):**
```js
// REMOVE this line:
app.use(cors({ origin: 'http://localhost:3000', credentials: true }));

// REPLACE with:
const ALLOWED_ORIGINS = [
  'http://localhost:3000',
  'http://localhost:5000',
  'file://',
  null,
];
app.use(cors({
  origin: (origin, callback) => {
    if (!origin) return callback(null, true);
    if (ALLOWED_ORIGINS.some(o => origin.startsWith(o))) return callback(null, true);
    callback(new Error(`CORS: ${origin} not allowed`));
  },
  credentials: true,
}));
```

**Change 2 — Port (at the bottom of server.js):**
```js
// REMOVE:
server.listen(5000, () => console.log('Server running on port 5000'));

// REPLACE with:
const PORT = process.env.PORT || 5000;
server.listen(PORT, () => console.log(`Server running on port ${PORT}`));
```

**Change 3 — Socket.IO CORS (around line 14):**
```js
// REMOVE:
const io = new Server(server, {
  cors: { origin: 'http://localhost:3000', methods: ['GET', 'POST'], credentials: true }
});

// REPLACE with:
const io = new Server(server, {
  cors: {
    origin: (origin, cb) => {
      if (!origin || ALLOWED_ORIGINS.some(o => origin.startsWith(o))) return cb(null, true);
      cb(new Error('socket CORS'));
    },
    methods: ['GET', 'POST'],
    credentials: true,
  }
});
```

### Step 3 — Add api.js to your React frontend

Create `frontend/src/api.js` with the content from `docs/frontend-api-patch.js`.

Then update your page files to use this instead of raw axios:

```js
// In every page/component that uses axios:

// OLD WAY (remove these):
import axios from 'axios';
const res = await axios.get('/api/courses');

// NEW WAY:
import api from '../api';           // adjust path as needed
const res = await api.get('/api/courses');
```

> **TIP:** Use VSCode's Find & Replace (Ctrl+Shift+H) to bulk-replace across all files.
> Find: `axios.get(` → Replace: `api.get(`
> Also replace `axios.post(`, `axios.put(`, `axios.delete(`, `axios.patch(`

### Step 6 — Build .exe Installers

**Option A — All three at once (recommended):**
```powershell
# In PowerShell, from lms-app/ root:
.\scripts\build-windows.ps1
```

**Option B — Build individually:**
```bash

cd frontend
npm run build
cd ..

# Then build each Electron app:
cd electron-wrapper/admin
npm install
npm run build:win
# Output: electron-wrapper/admin/dist/LMS-Admin-Setup-1.0.0.exe

npm install
npm run build:win
# Output: electron-wrapper/trainer/dist/LMS-Trainer-Setup-1.0.0.exe

cd ../trainee
npm install
npm run build:win
# Output: electron-wrapper/trainee/dist/LMS-Trainee-Setup-1.0.0.exe
```

## PART 2 — ANDROID APK

### Step 1 — Update API URL for mobile

Edit `frontend/src/api.js`:
```js
export const API_BASE = isElectron
  ? 'http://localhost:5000'
  : (process.env.REACT_APP_API_URL || 'http://YOUR_SERVER_IP:5000');
//                                              ^^^^^^^^^^^^^^^^
//                                              Replace with your actual server IP
```

Your phone and server must be on the **same WiFi network**, OR deploy the backend
to a free hosting service (see Part 3 below).

---

### Step 2 — Build APK (Linux/Mac/WSL)

```bash
# Make script executable:
chmod +x scripts/build-android.sh

# Run:
bash scripts/build-android.sh
```

**Output:**
```
dist-android/
  LMS-Admin-v1.0.apk
  LMS-Trainer-v1.0.apk
  LMS-Trainee-v1.0.apk
```

---

### Step 2 (Windows alternative) — Build APK manually

```bash
# 1. Build React
cd frontend
npm install
set REACT_APP_API_URL=http://YOUR_SERVER_IP:5000
npm run build

# 2. Install Capacitor
npm install @capacitor/core @capacitor/cli @capacitor/android @capacitor/splash-screen @capacitor/status-bar

# 3. Copy admin config
copy ..\android\capacitor.config.admin.ts capacitor.config.ts

# 4. Add Android platform
npx cap init "LMS Admin" "com.lms.admin" --web-dir build
npx cap add android
npx cap sync android

# 5. Open in Android Studio and build
npx cap open android
# In Android Studio: Build → Build Bundle(s)/APK(s) → Build APK(s)
# APK appears in: android/app/build/outputs/apk/debug/app-debug.apk
```

Repeat steps 3–5 with `capacitor.config.trainer.ts` and `capacitor.config.trainee.ts`.

---

### Step 3 — Install APK on Android Phone

1. Connect phone via USB to PC
2. Copy APK file to phone storage
3. On phone: Settings → Security → Enable "Install unknown apps"
4. Open file manager on phone → tap the APK → Install
5. App appears in your app drawer!

OR share via:
- Google Drive (upload APK, open link on phone)
- WhatsApp (send to yourself)
- Direct cable transfer

## PART 3 — FREE DEPLOYMENT (Optional: for internet-accessible backend)

For the Android APK to work anywhere (not just local WiFi), deploy your backend free:

### Option A — Railway.app (Recommended, easiest)
1. Go to https://railway.app → Sign up free
2. Click "New Project" → "Deploy from GitHub repo"
3. Push your `backend/` folder to GitHub first
4. Railway auto-detects Node.js and deploys
5. Add environment variables (copy from your `.env` file)
6. Get your URL: `https://your-app.railway.app`
7. Update `REACT_APP_API_URL=https://your-app.railway.app` in frontend

### Option B — Render.com (Also free)
1. https://render.com → New → Web Service
2. Connect GitHub repo
3. Build command: `npm install`
4. Start command: `node server.js`
5. Add env vars from your `.env`

### MongoDB — Keep using MongoDB Atlas (free tier)
Your existing MongoDB Atlas connection in `.env` works fine for deployment.

## COMMON ISSUES & FIXES

### "Windows protected your PC" warning
This appears because the .exe is not code-signed.
→ Tell users: click "More info" → "Run anyway"
→ For production: get a free code signing cert from https://signpath.io/

### APK "App not installed" error
→ Make sure "Install unknown apps" is enabled in Android settings
→ Try: Settings → Apps → Special app access → Install unknown apps

### Electron window shows blank/white screen
→ Backend might not have started yet. Wait 3-5 seconds after launch.
→ Check that MongoDB Atlas allows connections from anywhere (0.0.0.0/0)

### "Cannot connect to server" in Electron
→ The packaged app embeds the backend. If MongoDB connection fails:
  → Open your `.env` and ensure MONGO_URI is correct
  → The `.env` file must be in the `backend/` folder

### Different apps showing same content
→ Since all 3 apps load the same React app, role-based routing in App.js
  handles what each user sees based on their login role (admin/trainer/student).
  This is correct behavior.
